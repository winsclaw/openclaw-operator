# 访问与配对

## 通过 `install.sh` 部署的主网关

访问地址格式：

```text
https://<ingress-host>/?token=<OPENCLAW_GATEWAY_TOKEN>
```

如果页面提示 `pairing required`，先查看待审批请求：

```bash
kubectl exec -n aiops-openclaw deployment/openclaw -c main -- \
  node dist/index.js devices list
```

再批准指定请求：

```bash
kubectl exec -n aiops-openclaw deployment/openclaw -c main -- \
  node dist/index.js devices approve <REQUEST_ID>
```

## 通过 `install-instance.sh` 部署的实例节点

本地端口转发访问：

```bash
kubectl port-forward -n openclaw-node svc/<instance-name> 18789:18789
```

然后打开：

```text
http://localhost:18789
```

如果启用了 Ingress，也可以直接访问：

```text
https://<instance-ingress-host>/?token=<OPENCLAW_GATEWAY_TOKEN>
```

首次访问时看到 `pairing required` 是正常现象，需要审批当前浏览器的设备配对请求：

```bash
kubectl exec -n openclaw-node deployment/<instance-name> -c main -- \
  node dist/index.js devices list

kubectl exec -n openclaw-node deployment/<instance-name> -c main -- \
  node dist/index.js devices approve <REQUEST_ID>
```

审批完成后刷新页面即可。
